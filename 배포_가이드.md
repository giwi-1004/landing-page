## 1. Supabase — 신청자 DB 만들기

### 1-1. 프로젝트 생성

1. [Supabase Dashboard](https://supabase.com/dashboard) → **New project**
2. 리전: **Northeast Asia (Seoul)** 권장
3. DB 비밀번호 저장해 두기

### 1-2. 테이블 생성 (SQL 한 번 실행)

1. Supabase → **SQL Editor** → **New query**
2. 이 파일 전체를 붙여넣고 **Run**:

```
supabase/setup_lead_requests_full.sql
```

생성되는 테이블: **`public.lead_requests`**

| 컬럼 | 설명 |
|------|------|
| `id` | UUID (자동) |
| `name` | 이름 |
| `phone` | 전화번호 |
| `created_at` | 저장 시각 (UTC) |
| `created_at_kst` | 한국 시각 (표시용) |

### 1-3. API 키 복사

**Project Settings → API** 에서 아래 3개를 복사:

| 키 | 용도 |
|----|------|
| Project URL | `SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_URL` |
| anon public | `NEXT_PUBLIC_SUPABASE_ANON_KEY` |
| service_role (secret) | `SUPABASE_SERVICE_ROLE_KEY` — **절대 공개 금지** |

## 5. Vercel 배포

1. [vercel.com](https://vercel.com) → **Add New → Project**
2. GitHub 저장소 **Import**
3. 설정 확인:
   - **Framework:** Next.js
   - **Root Directory:** `./`
   - **Build Command:** `npm run build`
   - **Install Command:** `npm ci` (`vercel.json` 참고)
4. **Environment Variables** — 2-2 항목 등록
5. **Deploy**

배포 URL 예: `https://your-project.vercel.app`
